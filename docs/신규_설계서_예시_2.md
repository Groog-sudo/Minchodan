# 📱 VIP Assistant AI - 최적화 장애물 탐지 시스템 설계서 (Optimized Obstacle Detection System Design Specification)

본 설계서는 시각장애인의 안전 보행을 보조하기 위해 주변 위험 장애물을 실시간으로 인지하고 차별화된 청각/햅틱 경보를 전송하는 **온디바이스 최적화 장애물 탐지 및 위험 경보 시스템**의 상세 기술 명세를 정의합니다. (본 설계서에는 객체 탐지 서비스 집중화를 위해 TMAP 길안내 및 나침각/GPS 기반 내비게이션 기능이 완전히 제외되었습니다.)

## 🏗️ 1. 핵심 최적화 결정 (Core Optimization Decisions)

본 설계는 모바일 단말의 제한적인 연산 한계와 네트워크 대기 시간을 극복하기 위해, 프레임 전송 및 음성 출력은 단말(Thin Client)에서 처리하고 객체 탐지 및 상태 추적은 GPU 서버(FastAPI)에서 분산 처리하는 구조를 지향합니다.

```
                                 [단말 카메라 캡처]
                                          │
                  ┌───────────────────────┴───────────────────────┐
                  ▼                                               ▼
     [반사 스트림: 8~10 FPS]                         [인지 스트림: 2 FPS]
  (차량/오토바이 등 Priority 1)                (노면 상태 및 Priority 2/3 장애물)
                  │                                               │
                  ▼                                               ▼
         [반사 경로 (Reflex)]                           [인지 경로 (Cognitive)]
   - RAG/LLM/실시간 TTS 바이패스                  - 로컬 RAG 검색 (< 50ms)
   - 1.2m 내 위급 및 돌진체 대상                 - 로컬 LLM (gemma4-e2b) 가이드 생성
   - 사전합성 음성클립 즉시 재생                  - 300ms 초과 시 fallback.py 즉시 전환
                  │                                               │
                  └───────────────────────┬───────────────────────┘
                                          ▼
                             [위험 오디오 스케줄러 (Scheduler)]
                               - 1.2m 위급 진입 시 즉각 비프음 송출
                               - 소음 레벨 감응형 자동 볼륨 제어
```

1. **이중 캡처 주기 (Dual-rate Capture) 도입**:
   * 시속 20km(5.5m/s) 이상으로 돌진하는 고위험 이동체(오토바이, 차량 등)를 2 FPS 단일 주기로 탐지하면 안전 반응 시간을 확보할 수 없습니다.
   * 이를 방지하기 위해 단말은 카메라를 **이중 타이머**로 제어하여 프레임을 송출합니다.
     * **반사 스트림 (Reflex Stream)**: **8~10 FPS**로 캡처하여 고위험 사물(`car`, `motorcycle`)만 빠르게 감지 (추론 루프 최소화로 지연 < 100ms 유지)
     * **인지 스트림 (Cognitive Stream)**: **2 FPS**로 캡처하여 고정 장애물 및 노면 상태 분석

2. **이중 경로 연산 (Dual-path Execution) 아키텍처 구현**:
   * 위험도에 따라 실시간 백엔드 처리 경로를 이중화하여 반응 속도와 지능형 피드백을 동시에 획득합니다.
     * **반사 경로 (Reflex Path)**: 1.2m 이내 위급 장애물 혹은 다가오는 인명 위협체 발생 시 적용. RAG/LLM 연산을 완전히 스킵(Bypass)하고 앱 내장 고정 클립 즉시 재생 (종단 지연 < 300ms)
     * **인지 경로 (Cognitive Path)**: 경미한 고정 장애물(2.0m 내) 및 노면 파손 분석 시 적용. 로컬 RAG 검색(< 50ms)과 로컬 LLM 가이드 생성(< 300ms)을 실행하여 동적이고 안전한 가이드를 제공

3. **LLM 타임아웃 세이프티 가드 및 룰기반 Fallback 결합**:
   * 인지 경로 실행 중 로컬 LLM의 추론이 **300ms를 초과**해 지연되거나 L3 Validator 검증 실패 시, 즉시 `fallback.py`의 룰 기반 사전 정의 딕셔너리 멘트로 스위칭하여 시스템 지연을 방어합니다.

4. **단일 스케줄러를 통한 오디오 쿨다운 관리**:
   * 센서에서 0.5초마다 사물이 들어올 때 동일 멘트가 겹치지 않도록 방어하는 오디오 스케줄러를 적용하여 청각 과부하(Auditory Overload)를 차단합니다. (마지막 발화 후 최소 3.0초 쿨다운 보장, 단 위급 상황 진입 시에는 즉시 바이패스하여 선점 출력)

## 🛠️ 3. 핵심 모듈 최적화 설계 (Detailed Component Design)

### 3.1 이중 카메라 스트림 및 프레임 처리
* **동작**: 단말은 배터리 및 대역폭 효율 극대화를 위해 이중 주기 타이머를 실행합니다.
  * **Reflex 스트림**: 8~10 FPS 속도로 캡처하여 고위험 사물만 탐지. 지연 방지를 위해 리사이징만 적용 후 즉시 추론.
  * **Cognitive 스트림**: 2 FPS 속도로 노면 분할 및 고정 장애물 정밀 탐색 프레임 전송.

### 3.2 뎁스 가드 & ByteTrack 추적
* **동적 임계치**:
  * 돌진형 hazard (`car`, `motorcycle`): 3.5m 안전 범위 적용
  * 고정형 obstacle (`bollard`, `stairs`): 2.0m 안전 범위 적용
  * 상호작용 landmarks (`chair`, `table`): 1.5m 정밀 탐색 범위 적용
* **추적 상태 캐싱**: ByteTrack의 `track_id`를 기준으로 속도와 사용자를 향해 다가오는 속성(`direction == "towards"`)을 판별해 Redis에 30초간 저장하여 일시적 튀는 오인식(Flickering)을 보정합니다.

### 3.3 우선순위 분류 및 이중 경로 제어 (`prioritizer.py`)
이중 게이트(Reflex/Surface) 판별 원칙에 따라 감지된 객체를 1차 분류 및 분기합니다.

```python
# server/detection/prioritizer.py
# -*- coding: utf-8 -*-

class SafetyPrioritizer:
    def __init__(self, danger_threshold=1.2):
        self.danger_threshold = danger_threshold
        self.high_risk_classes = {"car", "motorcycle", "bus", "truck"}

    def dispatch_route(self, detections, surface_detections):
        """
        감지된 객체 리스트를 판별하여 반사(Reflex) 경로 및 인지(Cognitive) 경로로 분기합니다.
        """
        # 1. 반사 경로(Reflex Path) 판별
        # - 고위험 이동체가 1.2m 이내로 들어왔거나, 다가오는 인명 위협인 경우
        # - P0 노면(파손 점자블록, 계단 등)이 발밑 바로 아래 검출된 경우
        for obj in detections:
            if obj["class_name"] in self.high_risk_classes:
                if obj["distance"] <= self.danger_threshold or obj.get("is_threat", False):
                    return "reflex", obj

        for surf in surface_detections:
            if surf["class_name"] in ["braille_damaged", "crosswalk", "caution"]:
                # 하단부 초근접 검출 시 즉각 반사 경로 처리
                if surf.get("centroid_y", 0) > 480: # 640 해상도 기준 하단 75% 영역
                    return "reflex", surf

        # 2. 일반 상황인 경우 인지 경로(Cognitive Path) 대상으로 우선순위 정렬
        # - 클래스 등급(1순위: 차량/사람, 2순위: 킥보드/볼라드, 3순위: 의자)과 거리 순 정렬
        cognitive_candidates = []
        for obj in detections:
            cognitive_candidates.append(obj)

        cognitive_candidates.sort(key=lambda x: (x["priority"], x["distance"]))

        if cognitive_candidates:
            return "cognitive", cognitive_candidates[0]

        return "safe", None
```

## ⚡ 4. 실시간 이중 경로 연산 명세 (Execution Path)

### 4.1 반사 경로 (Reflex Path): 초고속 즉시 알림
* **목표 지연**: **< 300ms**
* **동작**: RAG 데이터베이스 조회 및 로컬 LLM 호출을 완전히 건너뜁니다.
* **음성**: 실시간 음성 합성을 하지 않고, 앱 설치 시 동봉된 사전합성 고정 MP3 파일(예: `reflex_clips/high_front.mp3`)의 ID만 웹소켓으로 던져 즉각 오디오 플레이백을 실행하며 동시에 햅틱 진동을 울립니다.

### 4.2 인지 경로 (Cognitive Path): 스마트 수칙 안내
* **목표 지연**: **1~2Hz (500ms ~ 1000ms)**
* **동작**:
  1. `retriever.py`가 ChromaDB 벡터 DB에서 로컬 임베딩(`nomic-embed-text`)을 통해 50ms 내에 해당 사물과 연동된 '안전 수칙'을 검색합니다.
  2. `llm_router.py`가 gemma4-e2b 로컬 LLM을 호출하여 RAG 수칙을 컨텍스트로 주입하고, 20자 이내의 대피 문장을 비동기 생성합니다.
  3. 만약 LLM 응답이 **300ms를 초과**할 경우, 즉시 타임아웃 세이프티 가드가 동작해 `fallback.py`에서 지정된 정적 경고문으로 핫스왑합니다.

```python
# server/fallback.py
# -*- coding: utf-8 -*-

FALLBACK_RULES = {
    "motorcycle": "오토바이 정면 진입, 즉시 멈추고 오른쪽 대피",
    "car": "차량 정면 접근, 보행을 멈추고 안전 구역 대피",
    "scooter": "점자블록 위 킥보드 방치, 왼쪽 우회",
    "bollard": "볼라드 정면, 우측 우회",
    "stairs": "전방 계단 감지, 서행하며 단차 주의",
    "braille_damaged": "점자블록 파손 구간, 발밑 주의 서행",
    "crosswalk": "횡단보도 진입, 일시 정지 후 신호 대기",
    "chair": "앉을 수 있는 의자 우측 감지"
}

def get_fallback_guidance(class_name: str, distance: float) -> str:
    msg = FALLBACK_RULES.get(class_name, "장애물 주의")
    return f"{msg} ({distance:.1f}미터 앞)"
```

## 📡 5. 통신 메시지 규격 (Data Contract)

WebSocket을 통해 실시간으로 단말과 주고받는 직관적인 데이터 계약 명세입니다. (내비게이션용 위치 좌표 정보가 완전히 배제되었습니다.)

### 5.1 클라이언트 ➔ 서버 (Client ➔ Server Payload)
2 FPS 또는 8~10 FPS 주기로 모바일 카메라가 촬영한 Base64 프레임 및 마이크 센서 소음 수준만을 전송합니다.
```json
{
  "type": "sensors_stream",
  "event_id": "8f8c6b73-04a2-4a0b-9ef1-1828a2a4b89d",
  "device_id": "device_user_01",
  "frame_jpeg_b64": "/9j/4AAQSkZJRgABAQAAAQABAAD...",
  "ambient_noise_db": 58,
  "elapsed_time": 4.5
}
```

### 5.2 서버 ➔ 클라이언트 (Server ➔ Client Payload)
RAG/LLM 혹은 Fallback 분석 후 단말기로 피드백을 전달하는 초경량 결과 데이터 명세입니다.
```json
{
  "type": "guidance_response",
  "event_id": "8f8c6b73-04a2-4a0b-9ef1-1828a2a4b89d",
  "status": "active",
  "selected_target": {
    "class_name": "bollard",
    "track_id": 5,
    "distance": 1.5,
    "is_danger": false,
    "is_threat": false,
    "bbox": [120, 200, 80, 200]
  },
  "guidance": "볼라드 정면, 우측 우회 (1.5미터 앞)",
  "should_speak": true,
  "tts_volume": 0.7,
  "beep_trigger": false,
  "haptic_trigger": false
}
```
